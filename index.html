MailSearch

Application web

Interface de recherche mails internes

GrizzlyDev

Contexte : Stage de 2ème Année de BTS SIO dans l'entreprise GrizzlyDev

Mise en situation : Mr GUILLIEN, Dirigeant de l'entreprise a besoin d'un site web, simple et sobre, qui fonctionnera à la manière d'un moteur de recherche, pour tous les mails stockés sur les serveurs de l'entreprise

Spécification Techniques : Modéliser une interface d'interrogation d'index Responsive via ElasticSearch(ELS) une base de données NoSQL au format JSON, a manipulé via des requêtes AJAX, au moyen d'une application web HTML5, Bootstrap 4.1, Framework JQuery 3.3

Contraintes :
- Aucune connaissance sur les bases de données NoSQL et du fonctionnement des requêtes AJAX/JSON
- Première utilisation de Kibana l’interface de visualisations de données, ainsi que de Bootstrap 4.1
- Documentation importante pour ELS, Bootstrap 4.1

Démarche de réalisation proposée :
- Formulaire avec les champs "From" pour l’expéditeur, "To" le destinataire, "Date de début" et "Date de fin"
- Optimiser et adapter pour chaque cas de recherche (partielle ou exacte), les informations, éviter les dépendances
- Limiter le flux avec une pagination, une recherche sur la semaine si non déclarée
- Afficher les informations utiles, ranger dans un tableau puis, les plus détaillées dans une section cacher

Previous Next


Langages et Technologies utilisés

Bilan de la mission :
L’application web est basique, mais conforme pour tout ce qui est fonctionnalité de recherche avancée, et d’affichage lisible et simple. Pour ce qui est des informations détaillées depuis le lien « infos », la fonctionnalité a été implémenté mais l’event JQuery, n’a pas été finalisée.

RDSAdmin

Application informatique

Gestion des serveurs

GrizzlyDev

Contexte : Stage de 2ème Année de BTS SIO dans l'entreprise GrizzlyDev

Mise en situation : Mr GUILLIEN, Dirigeant de l'entreprise a besoin d'une application informatique, lui permettant d'avoir main prise sur les sessions connecté sur ces serveurs, qui fonctionnera à la manière de Windows Remote Desktop Manager, en plus simpliste afin d'alléger la bande passante des serveurs

Spécification Techniques : Représenter en WindowsForm un espace de visualisation des personnes connectées sur un ou plusieurs serveurs Bureau à distance (Windows RDS) en Windows Form responsive, avec VB.NET. L'utilisateur pourra déconnecter une session, actualiser le form, puis supprimer et ajouté un serveur, tout en disposant d'une sauvegarde des serveurs ajoutés, ainsi que l'heure de la derniere mise à jour. Le logiciel s'adaptera a un poste contenant un serveur local, et devra être compatible Windows Server 2008 à 2016

Contraintes :
- Travail de documentation sur le fonctionnement d’une infrastructure réseau, des packages réseau Windows Terminal Services, et du langage VB.NET non experimenté jusqu'ici
- Ré-assimilé les bases de la POO, manipuler un fichier configuration en XML, rendre les fonctions dynamique et ergonomique au possible
- Apprendre la responsivité en Windows Form

Démarche de réalisation proposée :
- Optimiser et adapter ces méthodes, afin de le rendre ergonomique
- Interface Windows Form qui accueillera les informations dans une DataGridView
- Utiliser le package Cassia, utilisant des fonctions similaire à Windows Terminal Services
- Déterminer les informations pertinentes à afficher(durée d'inactivité, adresse IP, état...)
- Les informations temporel comme la durée d'inactivité, seront formatés pour plus de lisibilité
- Sauvegardes des serveurs sur un fichier config XML

Previous Next


Langages et Technologies utilisés

Bilan de la mission :
Les fonctionnalités principales ont été implémenter. La suppression d’un serveur est incomplète, manque encore de connaissance sur la manipulation d’un fichier configuration en XML. Cette mission m’aura permis de retravailler la POO plus dans la pratique mais aussi dans la théorie bien qu’il me manque certaine mécanique d’algorithmie pour pouvoir m’en sortir.

- Mise en place d'un Web Service sur un serveur Apache à l'aide de script PHP en flux JSON

Spécification Techniques : L’évaluation d’une candidature est établit par un RH. Une note (de 0 à 5) pour chacun des critères d’évaluation est définis, il rédige un bref commentaire, et remplit d’un bonus/malus de - 10 à + 10 points le cas échéant, chaque fiche d’évaluation est datée et signée par l’évaluateur. La somme des notes multipliées par le coefficient donnera une note final. on affichera au format PDF, la liste des candidats avec leur note, leur moyenne et les 5 meilleurs

Contraintes :
- Manipuler beaucoup d'informations tout en modélisant l'application
- Se répartir les taches et réunir les compétences adéquate

Démarche de réalisation proposée :
- Modéliser plusieurs WindowsForm afin de respecter le processus d'évaluation
- Séparer les fonctionnalités avec les membres du groupe
- Découpage en sprint du projet
- Documentation Technique
- Elaboration d'une base de données qui accueillera les données de VideoRecrut
- Collaborer avec la base de données PgSQL via l'ERP ODOO pour les informations des employés RH
